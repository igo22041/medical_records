# Инструкция по установке

## Требования

- PHP 7.4 или выше
- MySQL 5.7 или выше (или MariaDB 10.2+)
- Веб-сервер (Apache/Nginx) или встроенный PHP сервер

## Шаги установки

### 1. Настройка базы данных

#### Вариант A: Через командную строку
```bash
mysql -u root -p < database/init.sql
```

#### Вариант B: Через phpMyAdmin
1. Откройте phpMyAdmin
2. Создайте новую базу данных `medical_records`
3. Выберите эту базу данных
4. Перейдите на вкладку "Импорт"
5. Выберите файл `database/init.sql` и нажмите "Вперед"

### 2. Настройка подключения к базе данных

Откройте файл `config/database.php` и измените параметры подключения:

```php
private $host = "localhost";        // Адрес сервера БД
private $db_name = "medical_records"; // Имя базы данных
private $username = "root";          // Имя пользователя БД
private $password = "";              // Пароль БД
```

### 3. Запуск приложения

#### Вариант A: Встроенный PHP сервер (для разработки)
```bash
php -S localhost:8000
```
Затем откройте в браузере: `http://localhost:8000`

#### Вариант B: Apache/Nginx
1. Скопируйте проект в директорию веб-сервера (например, `htdocs` или `www`)
2. Настройте виртуальный хост (если необходимо)
3. Откройте приложение в браузере

### 4. Первый вход

Используйте тестовые учетные данные:

**Администратор:**
- Имя пользователя: `admin`
- Пароль: `admin123`

**Пользователь:**
- Имя пользователя: `user`
- Пароль: `user123`

## Решение проблем

### Ошибка подключения к базе данных
- Проверьте, что MySQL сервер запущен
- Убедитесь, что параметры подключения в `config/database.php` правильные
- Проверьте, что база данных создана и таблицы импортированы

### Страница не загружается
- Убедитесь, что PHP установлен и работает
- Проверьте права доступа к файлам
- Проверьте логи веб-сервера

### Стили не применяются
- Проверьте, что файл `assets/css/style.css` существует
- Убедитесь, что пути к CSS файлам правильные
- Очистите кэш браузера

## Безопасность для продакшена

Перед развертыванием на продакшене:

1. Измените пароли администратора и тестового пользователя
2. Настройте безопасное подключение к БД
3. Включите HTTPS
4. Настройте правильные права доступа к файлам
5. Отключите отображение ошибок PHP в `php.ini`:
   ```ini
   display_errors = Off
   error_reporting = E_ALL
   log_errors = On
   ```
